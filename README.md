# Database Mastery

## Goal

This repository is a structured learning system for mastering databases, from fundamentals to advanced system design.

It is designed to:

- Build deep understanding of database internals
- Practice real-world database engineering concepts
- Track learning progress in a structured way
- Serve as a long-term knowledge base

---

## Repository Structure

- fundamentals/             -> Core database concepts 
- sql/                      -> SQL deep dive 
    - basics/               -> Basic queries, filtering 
    - joins/                -> All join types and use cases 
    - aggregation/          -> Grouping, aggregates 
    - advanced/             -> Window functions, CTEs 
    
- indexing/                 -> Index types and strategies 
- optimization/             -> Query performance tuning 
- transactions/             -> ACID, isolation levels 
- scaling/                  -> Sharding, replication 
- nosql/                    -> NoSQL databases and trade-offs 
- system-design/            -> Real-world architecture patterns 
- projects/                 -> Hands-on implementations 
- notes/                    -> Miscellaneous learnings

---

## Learning System Rules

### One Issue = One Concept

Each GitHub issue represents a single concept to learn.

---

### One Concept = One File

Each concept is documented as a single `.md` file.

Examples:

- `sql/joins/inner-join.md`
- `indexing/b-tree-index.md`
- `transactions/isolation-levels.md`

---

## Naming Convention

- Use lowercase
- Use hyphens instead of spaces

Examples:

- ✅ `inner-join.md`
- ❌ `InnerJoin.md`

---

## Issue -> File Mapping

Each issue must result in:

- A note file (for concepts), OR
- A project folder (for practical implementations)

---

## Outcome

By completing this repository, you will:

- Understand how databases actually work internally
- Design scalable and efficient data systems
- Be able to handle real-world backend challenges
- Build strong system design intuition

---

## Work in Progress

This is a living knowledge base, continuously updated as learning progresses.
